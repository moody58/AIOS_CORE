# 1. Scopo del Radar

Il Radar AIOS è il sistema di osservazione strategica e incubazione
delle idee.

Serve a:\
• raccogliere idee e intuizioni non ancora strutturate\
• evitare la dispersione di spunti nelle chat\
• mantenere visibile il potenziale evolutivo del sistema\
• creare un ponte tra intuizione e progetto operativo

Il Radar non è un luogo decisionale ma uno spazio di osservazione.

# 2. Principio di funzionamento

Il Radar accoglie elementi che non sono ancora pronti per diventare
progetti.

Questi elementi possono essere:\
• idee di nuovi progetti\
• opportunità strategiche\
• miglioramenti del sistema\
• strumenti o tecnologie da esplorare

Ogni elemento del radar deve essere descritto in forma sintetica.

# 3. Struttura delle voci Radar

Ogni voce inserita nel Radar include:

• Nome idea\
• Breve descrizione\
• Possibile ambito (business, sistema, strumento, ricerca)\
• Stato di maturazione\
• Note eventuali

Le voci devono restare brevi per mantenere il radar leggibile.

# 4. Stati delle idee

Le idee nel radar possono attraversare diversi stati:

Segnalata\
Idea appena annotata.

Osservazione\
Idea monitorata ma non ancora sviluppata.

Valutazione\
Idea in analisi per possibile sviluppo.

Strutturazione\
Idea in fase di trasformazione in progetto.

# 5. Passaggio Radar → Progetto

Quando un\'idea diventa sufficientemente chiara può trasformarsi in
progetto.

Flusso:

Idea Radar\
↓\
Valutazione\
↓\
Strutturazione\
↓\
Nuovo progetto

Il nuovo progetto verrà registrato nel Project Index AIOS.

# 6. Regole operative

• il radar non deve diventare un archivio caotico\
• le voci devono restare sintetiche\
• idee duplicate devono essere unificate\
• quando un\'idea diventa concreta deve uscire dal radar

Il radar serve per osservare, non per accumulare.
